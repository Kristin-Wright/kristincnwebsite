    In line with the requirements of Unit 2, Outcome 1, I prepared appropriate content for my site by responding to each of the questions posed in the assessment brief. In response to Unit 2, Outcome 2, I created and edited all written content in a Google Doc file saved on Google Drive, then pasting and styling in VS Code. Images and PDFs were collected locally on my Mac computer (running Mac OS Catalina 10.15.4) in the root folder of my site, retrieved via VS Code and imported into the project.

    For assessment criteria 2.3 - selecting and using appropriate editing and formatting techniques for clarity and good navigation, I referred to No.4 in Jakob Nielsen’s “10 Usability Heuristics for User Interface Design”[1] - “Consistency and Standards” - which states that “Users should not have to wonder whether different words, situations, or actions mean the same thing. Follow platform conventions” and that “consistency is one of the strongest contributors to usability”. In this vein, “A standard ensures that your users can understand the individual interface elements in your design and that they know where to look for what features”.

    With this in mind, I followed user interface convention and placed my logo in the top left hand corner, linked it back to the homepage (to satisfy heuristic principle No. 3 “User Control and Freedom”, where users should always be given an ‘escape’ or ‘exit’ option) and made sure that the navigation bar was consistently positioned at the top of each page through CSS styling of the .navbar class with ‘position: fixed’. This ensured that the linked logo is always where the user expects to find it.

    In order to achieve assessment criteria 2.4, I coded relative and absolute links to both internal files and external websites respectively within the HTML and CSS site files. These included creating local links between pages within the website (for example, linking <a href="index.html"> within the logo in the nav bar allows the user to return to the homepage), and also to the CSS stylesheet with a <link rel="stylesheet" type="text/css" href="main.css"> link in the head of each webpage.

    Google Fonts were in turn imported into the CSS stylesheet with, for example, ‘@import url(https://fonts.googleapis.com/css?family=Open+Sans)’ and external websites can opened in a new tab from a page by the user clicking on a link with the target attribute, such as <a href=”https://www.w3schools.com/html/html_links.asp” target="_blank">HTML Links.</a>.

    In terms of images for 2.5, I selected web-optimised PNGs over JPEGs if a transparent background was required (for example, in the logo) and SVGs (Scalable Vector Graphics) over PNGs if the website needed to be resized responsively to accommodate different screen sizes without pixelation or loss of quality. There was otherwise no need to change file formats to display content correctly.

    Finally, to demonstrate attainment of assessment criteria 2.6 I tested each page of the website by opening up the preview in the browser to check that visual updates made in VS code displayed correctly, and clicked through each link to ensure that none were broken. Invaluable tutor feedback via screen sharing on Zoom also helped to resolve a few persistent issues with the nav bar styling and global properties inherited from the universal selector.

    Sadly not all the bugs have been squashed yet (the nav bar styling needs further work so that it doesn't overlay the body text when scrolling, and spacing/positioning of the elements on each of the pages desperately needs attention amongst other issues), but I’ll continue to test and seek feedback to make the site the best it can be.
